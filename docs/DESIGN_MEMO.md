# Volatility Cascade — Design Memo

**Date:** 2026-07-13 | **Authors:** Nitya Hapani (lead), pong (implementation)  | **Repo:** https://github.com/syndatakit/volcascade | **Upstream:** https://github.com/Nityahapani/volcascade
**Target:** Modern Finance (DOAJ peer-reviewed)

---

## Core Objective

Build a Python package + peer-reviewed paper defining the **volatility cascade** — a term structure of differentiation order for realized volatility — to:

1. **H1:** Classify regime breaks
2. **H2:** Mark regime exits
3. **H3:** Separate idiosyncratic from systemic risk via cross-sectional decoupling
4. **H4:** Test dynamics in frontier/emerging markets

**Distinguishing features:** Shape-based, OHLCV-only, options-free; first cross-market extension to frontier markets.

---

## Locked Methodology Decisions

### Cascade Construction & Analysis

| Decision | Choice | Rationale |
|---|---|---|
| **Cascade orders** | 1–4 of realized volatility, z-scored against trailing history | Anchored by "vol is rough" literature (Gatheral et al. 2018); orders beyond 4 become noise-dominated |
| **Lookback windows** | 60, 120, 252 trading days | Captures day-of-week effects (Abdeldayem 2025) and VVIX asymmetry (Albers 2023) |
| **Slope summary** | Linear regression slope of order index vs z-score (primary); entropy of cascade distribution (secondary) | Parsimonious + reviewable; entropy as non-linear robustness check |
| **H3 decoupling** | Chow test on joint distribution at each order k (primary); cross-correlation threshold (robustness) | Most defensible against "this is just smoothing" critique; BH-FDR for multiple testing |
| **Comparison battery** | RCM (Ang-Bekaert 2002b), Bai-Perron F-test, HMM (Gaussian 2-/3-state), MS-AR(1) (Hamilton 1989), Wasserstein k-means (Campani 2021), Inclán-Tiao CUSUM | Canonical set across regime-detection literature |
| **Adversarial test** | Synthetic AR/GARCH series calibrated to DM and FM liquidity params; verify no spurious cascade steepening | **Strongest piece of paper** — rebuts "this is just vol-of-vol with extra steps" |
| **Multiple testing** | Benjamini-Hochberg FDR across all tests (4 hypotheses × 2 samples × 3 lookbacks × 5 forward windows) | Preempts reviewer concerns; defines unit of "significance" |
| **Sample transparency** | Effective N at each differentiation order reported explicitly; order-4 frontier estimates marked exploratory | Transparency about where data carries information |

### Pre-Registered Success Criteria

| Hypothesis | Pass Criterion |
|---|---|
| **H1** | Cascade-classified events → ≥2× larger forward 5-day drawdown than order-1-only classification |
| **H2** | Cascade-convergence exits → <30% false exit rate vs. ~50% for order-1 MA baseline |
| **H3** | Decoupling order predicts event type with AUC > 0.7 |
| **H4** | Classification accuracy differs significantly between DM and FM (p < 0.05 after BH-FDR) |

**Purpose:** Prevents p-hacking; gives reviewers clean pass/fail.

## Scope & Samples

### Paper Coverage
- **All 4 hypotheses** across two universes:
  - **Developed (DM):** S&P 500 constituents + SPDR sector ETFs
  - **Frontier (FM):** 6 country ETFs + constituents (NSE Kenya, GSE Ghana, BVSP Brazil, JSE South Africa, NSE India, BSE Bangladesh)

### Package Rollout
- **MVP (v0.1.0):** Core cascade construction + H3 decoupling + 3 baselines (HMM, Wasserstein k-means, Inclán-Tiao CUSUM)
- **v0.2.0+:** Bai-Perron, RCM, additional decoupling methods, full viz suite
- **Pilot sequence:** S&P 500 first (faster iteration), then frontier extension

### H3 Ground-Truth Event Classification

| Class | Sources | Examples |
|---|---|---|
| **Idiosyncratic** | Zacks earnings surprises, SDC M&A, BioPharmCatalyst FDA, CEO exits | Single-name earnings beats/misses, deal announcements, binary trial readouts |
| **Systemic** | FOMC, NFP, GFC peak days, COVID crash | Macro-policy events, broad-market shocks |

**Canonical source:** `data/ground_truth_events.csv` (public sources, curated table)

### Strategic Positioning

**H1-primary, H3-differentiator, H4 as cross-market distinguishing feature** vs. existing vol-of-vol literature. H1 is intuitive entry point for reviewers; H3 is most novel; H4 is unique contribution.

## Implementation Notes (2026-07-14)

**Known deltas between memo and current code:**

1. **H3 decoupling (`volcascade.decoupling`)** — Per-order decoupling test (z^(k) for k=1..4 separately) not yet implemented. Current `chow_decoupling()` operates on single z-scored series across all k ("flat cascade" baseline). H3 v3–v5 results generated by calling `chow_statistic` on order-specific z^(k) directly in experiment scripts, not via package API. See IMPLEMENTATION_NOTES.md §1.

2. **Baseline simplifications:**
   - `wasserstein_regime` uses Euclidean KMeans on histogram features, not true Wasserstein-2 distance (Campani 2021)
   - `bai_perron_breaks` runs PELT with RBF kernel, not Bai-Perron's sequential F-test
   - See IMPLEMENTATION_NOTES.md §2

3. **Frontier data layer** — `volcascade.io.FRONTIER_SAMPLE` declares 6 country entries (NSE_KE, GSE_GH, BVSP, JSE, NSE_IN, BSE_BD), but 4 of them reuse EZA (iShares MSCI South Africa) as the proxy because there is no direct Kenya, Ghana, or Brazil-index ETF with sufficient data; BSE_BD uses XBS which has no coverage. The empirical sample is effectively 3 frontier ETFs (EZA, EWZ, INDA).

4. **Pre-reg parameter consistency** — All headline scripts use the pre-reg params (orders=1-4, inner_window=10, zscore_lookback=120, forward_days=5). The package defaults (`build(inner_window=20)`, `zscore(lookback=252)`) differ; every script overrides the defaults explicitly. The pilot_spy.py script uses inner_window=20 — the only pre-reg-inconsistent experiment, written before the pre-reg was finalized.

5. **API export gap** — `volcascade.baselines` defines `bai_perron_breaks` and `rcm` in its `__all__` but `volcascade.__init__` does not re-export them. `from volcascade import bai_perron_breaks` fails with ImportError. Fix: add to `__init__.py`'s imports and `__all__`.

6. **pyproject.toml metadata** — Repository URL still points at the upstream `Nityahapani/volcascade` rather than the fork `syndatakit/volcascade`. Authors list is Nitya-only; the README credits both Nitya and pong. Fix in a follow-up commit.
